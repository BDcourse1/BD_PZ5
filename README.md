# Приложение для поиска каллорийности продуктов

Приложение для поиска калорийности продуктов через API OpenFoodFacts с графическим интерфейсом на PyQt5.

## Функциональность

- **Поиск по штрихкоду** - точный поиск продукта по штрихкоду
- **Поиск по названию** - поиск продуктов по названию с отображением нескольких результатов
- **Отображение nutritional информации**:
  - Калории на 100г
  - Белки на 100г
  - Жиры на 100г
  - Углеводы на 100г
  - Информация о порции
- **Удобный интерфейс** с вкладками для разных типов поиска
- **Асинхронные запросы** - не блокирует интерфейс во время поиска
- **Детальная информация** - просмотр полной информации о продукте по двойному клику

## Установка и запуск

### Требования

- Python 3.7+
- Установленные зависимости из `requirements.txt`

### Установка

1. Клонируйте репозиторий:
```bash
git clone <url-репозитория>
cd calorie-finder-app
```
2. Установите зависимости:
```bash
pip install -r requirements.txt
```
3. Запустите приложение:
 ```bash
python main2.py
```

## Зависимости
**Основные зависимости:**

- PyQt5 - для графического интерфейса
- requests - для HTTP-запросов к API

## Использование

### Поиск по штрихкоду:
1. Перейдите на вкладку "Поиск по штрихкоду"
2. Введите штрихкод продукта в поле ввода
3. Нажмите "Найти" или Enter
4. Результат отобразится в таблице ниже

### Поиск по названию:
1. Перейдите на вкладку "Поиск по названию"
2. Введите название продукта (например, "шоколад", "молоко")
3. Нажмите "Найти" или Enter
4. Список найденных продуктов отобразится в таблице
5. Для просмотра детальной информации дважды кликните на продукт

## Особенности реализации
- Многопоточность: Поиск выполняется в отдельном потоке, не блокируя UI
- Обработка ошибок: Информативные сообщения об ошибках сети и API
- Интернационализация: Поиск настроен на русскоязычные данные
- Progress индикаторы: Визуальная индикация процесса поиска

## API
**Приложение использует OpenFoodFacts API v2:**
- GET /api/v2/product/{barcode} - получение продукта по штрихкоду
- GET /api/v2/search - поиск продуктов по названию

## Разработка
### Основные компоненты:
1. CalorieFinderApp - главный класс приложения
2. SearchWorker - worker для выполнения поиска в отдельном потоке
3. extract_kcal() - функция извлечения nutritional данных
4. get_product_by_barcode() - запрос к API по штрихкоду
5. search_products() - поиск продуктов по названию
